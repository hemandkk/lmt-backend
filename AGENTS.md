# AGENTS.md — lmt-backend

FastAPI backend for a Lead Management & Training (LMT) system. PostgreSQL via SQLAlchemy + Alembic.

## Run locally

```bash
.venv\Scripts\activate
pip install -r requirements.txt
alembic upgrade head
uvicorn main:app --reload
```

Dev server at `http://localhost:8000`. Default admin seeded on startup: `admin@example.com` / `admin123`.

## Migrations (read this first)

Every model change **must** produce an Alembic migration. Do NOT rely on `Base.metadata.create_all()` — it does not add columns to existing tables.

```bash
alembic revision --autogenerate -m "description"
alembic upgrade head
alembic current          # verify applied version
```

`main.py` also has `ensure_schema_updates()` with raw ALTER TABLE patches — these are legacy stopgaps, not a migration strategy. New schema changes go in Alembic.

When adding a new model, import it in `alembic/env.py` so autogenerate picks it up.

## Architecture

```
main.py                          # FastAPI app factory, startup seeds, CORS, mount /uploads
app/
  api/v1/
    api.py                       # APIRouter aggregating all sub-routers
    auth.py, prospects.py, ...   # Route handlers
  services/                      # Business logic (static methods, no FastAPI deps)
  repositories/                  # SQLAlchemy query layer (static methods)
  schemas/                       # Pydantic request/response models
  db/
    models/                      # SQLAlchemy ORM models (inherit TimestampMixin + Base)
    session.py                   # engine + SessionLocal + get_db dependency
    base.py                      # DeclarativeBase
    mixins.py                    # TimestampMixin (created_at, updated_at)
  core/
    config.py                    # Pydantic Settings — reads .env
    security.py                  # JWT encode/decode, bcrypt hash/verify
    id_generator.py              # Sequential code generator (PRO00001, PAY00001, …)
    file_storage.py              # Upload file save/replace/delete
  dependencies/
    auth.py                      # get_current_user / get_optional_user (JWT)
    permissions.py               # require_admin, ensure_prospect_access, etc.
  services/google_sheets_service.py  # Optional Google Sheets lead sync
```

All routes are mounted under `/api/v1` (except payment_router which mounts directly).

## Key conventions

- **Layering**: Routes → Services → Repositories. Services and repositories use `@staticmethod` methods (no class instantiation). Routes depend on `get_current_user` from `app.dependencies.auth`.
- **Employee scope**: Employees see only their own leads (`assigned_to_id == user.id`). Admins see all. This is enforced in route handlers via `_employee_scope_id()` / `require_admin`.
- **Prospect IDs**: Auto-generated sequential codes like `PRO00001`. Generated by `app.core.id_generator.generate_next_code()`. Same pattern for payments (`PAY`), documents (`DOC`), employees (`EMP`), courses (`CRS`).
- **TimestampMixin**: All models inherit from it — `created_at` and `updated_at` are server-managed (PostgreSQL `now()`).
- **Multipart forms**: The prospects endpoint accepts both `application/json` and `multipart/form-data`. The multipart path has its own parsing logic (`_parse_multipart_lead`) supporting flat fields, `payload` JSON strings, and `documents[]`/`docTypes[]` paired arrays.
- **Google Sheets sync**: Optional (controlled by `GOOGLE_SHEETS_ENABLED` in `.env`). Leads are auto-synced to a Google Sheet on create/update. See `.env.google-sheets.example` for required env vars. Credentials go in `app/credentials/`.
- **File uploads**: `FileStorage` supports `STORAGE_BACKEND=local` (`app/uploads/` + `/uploads` mount) or `s3` (S3/Cloudflare R2). Deletes/replaces also remove the object from storage. With `s3`, DB stores absolute public URLs (`S3_PUBLIC_BASE_URL`). `APP_BASE_URL` still prefixes relative `/uploads/...` links in exports/sheets.

## Environment

Required `.env` keys: `DATABASE_URL`, `JWT_SECRET_KEY`, `JWT_EXPIRE_MINUTES`. See `.env` and `.env.google-sheets.example` for the full list.

Database: PostgreSQL with `psycopg` driver. Connection string format: `postgresql+psycopg://user:pass@host/dbname`.

## Testing

`pytest` and `httpx` are in requirements.txt. No test files exist yet. When adding tests:
- Use `httpx.AsyncClient` with the FastAPI `TestClient` pattern.
- Mock the database session via `app.db.session.get_db` dependency override.
